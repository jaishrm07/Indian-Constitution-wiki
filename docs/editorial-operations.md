# Editorial Operations

This document defines how the project stays current without losing source discipline. The site is a constitutional reference system first, so operations must protect accuracy before breadth.

## Operating Principles

1. Constitutional text is the anchor for every interpretive page.
2. Official institutional records come before secondary explanations.
3. Current affairs are only worth publishing when they have a constitutional hook.
4. Research notes can be exploratory; published content must be reviewed and source-controlled.
5. Coverage should grow in clusters, not as isolated pages.

## Cadence

### Daily

- scan official sources for new constitutional developments
- triage new bills, judgments, notifications, and institutional disputes
- update or create current-affairs issue notes only when an item is constitutionally relevant
- record new source captures in the research log

### Weekly

- review all ongoing current-affairs pages
- verify whether any issue should move from `ongoing` to `resolved` or `archived`
- check source links for stale or redirected official URLs
- compare canonical coverage against the research vault and fill gaps

### Monthly

- review doctrine clusters and identify weak topic pages
- expand article, case, and timeline coverage where live issues are concentrated
- audit source tiers and retire weak secondary sources when a better official or structured source exists
- review learnings and turn repeated operational lessons into repo guidance

### Quarterly

- archive old live issues that no longer need active monitoring
- review the source registry for overlap, duplication, and broken references
- refresh topical landing pages and editorial priorities
- check that the canonical graph still matches the intended product boundaries

## Current-Affairs SOP

Use this sequence for any new live issue:

1. Confirm constitutional relevance.
2. Identify the governing Articles, Parts, Schedules, and doctrines.
3. Collect at least one official source and one primary institutional record.
4. Add secondary trackers only for chronology or readability.
5. Write the issue note with a clear constitutional question, status, and what to watch next.
6. Link the issue to articles, cases, topics, institutions, and source records.
7. Set the review date and follow-up cadence.

Do not publish a current-affairs note if:

- there is no constitutional anchor
- the only sources are social posts or commentary
- the issue is still too vague to describe in constitutional terms

## Source Review SOP

Use this sequence for any new source or source refresh:

1. identify the source tier
2. record the upstream URL and publication date
3. check whether it is official, judicial, legislative, or secondary
4. determine whether it should be added to the source registry
5. note where the source is used across Articles, cases, topics, or issues
6. record any access limitations or PDF-specific quirks

Every source review should answer:

- what does this source prove
- what does it not prove
- what should never be inferred from it

## Editorial Review Checklist

Before merging any substantive content or documentation update, confirm:

- every live issue has at least one official or institutional source
- every interpretive page points back to constitutional text
- every status value is current
- every source URL still resolves
- any new workflow is reflected in `learnings.md`

## Escalation Rules

Escalate immediately when:

- an official source contradicts a published summary
- a constitutional claim cannot be traced to source text
- a live issue page has drifted into commentary without citations
- an archive or redirect breaks the trust chain for a cited source

## Maintainership Standard

This project should feel like a maintained reference desk, not a static content dump. If a page cannot be kept current, it should be archived or narrowed until it can.
