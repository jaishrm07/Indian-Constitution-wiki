# Knowledge Vault

This is the Obsidian-compatible research graph for the project.

Purpose:

- explore relationships before publishing
- maintain evolving notes on constitutional entities and issues
- keep research, synthesis, and editorial planning out of chat history

This vault is not the canonical publishing database. The website should still publish from structured content in `src/content/`.

Main files:

- `index.md`: navigational catalog of durable notes and hubs
- `log.md`: append-only record of ingests, reviews, and maintenance work
- `templates/`: note templates for common entities and workflows

Recommended workflow:

1. capture or reference a source in `raw/`
2. create or update a note here
3. connect it to related Articles, Cases, Topics, Amendments, and institutions
4. promote reviewed knowledge into `src/content/`
